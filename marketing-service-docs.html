Executive Summary

This document outlines the operational framework for delivering automated digital marketing services to small and medium businesses (SMBs). The service enables customers to establish and maintain a consistent social media presence with minimal ongoing effort through AI-powered content generation and scheduled publishing.

Core Value Proposition

Customers complete a one-time setup session, after which automated systems generate and publish content on their behalf. The service targets businesses that:

  • Lack time or expertise for social media management
  • Want digital presence without dedicated marketing staff
  • Prefer predictable costs over hiring or agency fees
  • Need visibility for local SEO/AEO (dentists, vets, trades, etc.)
ðŸ’¡ Key Differentiator

This is not a software product sold to customers. It is a managed service where we handle the complexity of tools, configuration, and publishing. Customers pay for outcomes (posts published, presence maintained), not for tool access.

Success Metrics

  • Activation Rate: % of signed customers who complete onboarding within 7 days
  • Content Delivery Rate: % of scheduled posts published on time
  • Customer Retention: Monthly churn rate target <5%
  • Quality Score: % of posts requiring no human intervention

  • Maintain backup tool evaluation
  • Export customer configurations regularly
  • Monitor tool vendor health/funding

Design Principles

  • Tool Agnosticism: Abstract tool-specific logic so we can swap vendors without rewriting workflows
  • Fail-Safe Defaults: If a system component fails, default to "don't publish" rather than publishing potentially broken content
  • Audit Trail: Log all content generation, reviews, and publishing events for debugging and compliance
  • Graceful Degradation: If video generation fails, fall back to image-only post rather than missing the slot entirely

Blotato provides a community N8N node that handles TikTok authentication and posting. Key advantages:

The Blotato N8N node is a community node that only works on self-hosted N8N instances. N8N Cloud does not support community nodes, so the infrastructure must be self-hosted.

Direct API Alternative

  1. Register as a TikTok for Developers partner
  2. Create an app and request Content Posting API access
  3. Complete TikTok's security audit (can take 2-4 weeks)
  4. Implement OAuth 2.0 PKCE flow for each customer
  5. Handle video upload via chunked transfer
â›” Not Recommended

Direct TikTok API implementation adds 40+ hours of development time and ongoing maintenance burden. Blotato at $29/month is significantly more cost-effective.

Token Management

  • Storage: Tokens encrypted at rest (AES-256) in dedicated token vault
  • Refresh: Automatic refresh before expiration (most tokens expire in 60-90 days)
  • Revocation Handling: If customer revokes access, detect on next API call â†’ pause publishing â†’ notify ops
  • Rotation: Refresh tokens rotated on each use where supported
# Token Refresh Logic def get_valid_token(customer_id, platform): token = vault.get_token(customer_id, platform) if token.expires_at < now() + timedelta(days=7): # Token expiring soon, refresh proactively try: new_token = platform_api.refresh(token.refresh_token) vault.store_token(customer_id, platform, new_token) return new_token except RefreshError: # Refresh failed - token likely revoked alert_ops(customer_id, platform, "Token refresh failed") raise TokenRevokedException() return token

  • Customer Data: Business info, preferences stored in encrypted database

ðŸ”´ Platform API Changes

Likelihood: Medium  |  Impact: High

Mitigation:

  • Use established scheduling tools (Buffer, etc.) that absorb API changes
  • Monitor platform developer changelogs
  • Maintain 2-week content buffer to handle outages
  • Have manual publishing fallback procedure documented
ðŸŸ¡ Rate Limiting

Risk: Hit platform rate limits during batch operations.

Likelihood: Medium  |  Impact: Medium

Mitigation:

  • Implement request queuing with backoff
  • Spread batch jobs across time (not all customers at once)
  • Monitor rate limit headers, throttle proactively
ðŸŸ¡ AI Tool Vendor Risk

Risk: Content generation tool changes pricing, quality, or shuts down.

Likelihood: Low-Medium  |  Impact: High

Mitigation:

  • Abstract tool interface so swapping is easier
  • Maintain evaluation notes on alternative tools
  • Export prompt templates in portable format
  • Consider building direct LLM API integration as backup

ðŸŸ¡ Content Generation Bottleneck
